# b install

Install binaries. If no arguments are given, `b` installs all binaries from the `b.yaml` configuration file. You can also install specific binaries on-the-fly.

## Usage

```bash
b install [binary...] [flags]
```

## Examples

### Install all binaries from config

If a `b.yaml` file is present, this command will install all the binaries listed in it.

```bash
b install
```

### Install a specific binary

Install the latest version of a specific binary.

```bash
b install jq
```

### Install a specific version

Append `@<version>` to the binary name to install a specific version.

```bash
b install jq@jq-1.7
```

### Install from GitLab

Use a full `gitlab.com` path to install from GitLab releases.

```bash
b install gitlab.com/owner/repo
```

### Install from Gitea / Codeberg

Use the full host path for Gitea or Forgejo instances like Codeberg.

```bash
b install codeberg.org/owner/repo
```

### Install from Go source

Prefix with `go://` to compile and install from Go module source. Requires `go` on your PATH.

```bash
b install go://github.com/jrhouston/tfk8s
```

### Install from Docker image

Prefix with `docker://` to extract a binary from a Docker/OCI image. Requires a container runtime (docker, podman, or nerdctl).

```bash
b install docker://hashicorp/terraform
```

### Install and add to config

Use the `--add` flag to install a binary and simultaneously add it to your `b.yaml`.

```bash
b install --add kubectl
```

### Force an installation

Use the `--force` flag to overwrite an existing binary.

```bash
b install --force jq
```

### Install with an alias

Use the `--alias` flag to install a binary under a different name. This is useful when you want to use a custom name for a binary while still using the original binary's implementation.

```bash
b install --alias envsubst renvsubst
```

This installs `renvsubst` but makes it available as `envsubst`. The alias binary will:
- Download and install the original binary (`renvsubst`)
- Create the alias binary file (`envsubst`) 
- Work with all commands like version detection and execution
- Show in listings with the alias relationship: `renvsubst (envsubst)`

You can combine `--alias` with other flags:

```bash
# Install with alias and add to config
b install --alias envsubst --add renvsubst

# Install specific version with alias
b install --alias myenvsubst renvsubst@v0.10.0
```

### Select a specific release asset

When a GitHub release contains multiple binaries (e.g., `argsh`, `argsh-so`, `minifier`, `shdoc`), `b` auto-detects the best match for your OS and architecture. If multiple assets match with the same score, `b` will prompt you interactively to choose one.

Use the `--asset` flag with a glob pattern to select a specific asset:

```bash
# Install the standalone argsh-so binary instead of the default
b install --asset "argsh-so-*" arg-sh/argsh

# Install the .so shared object
b install --asset "argsh-linux-*.so" arg-sh/argsh

# Install shdoc from the same release
b install --asset "shdoc-*" arg-sh/argsh
```

You can persist the asset filter in `b.yaml`:

```yaml
binaries:
  arg-sh/argsh:
    asset: "argsh-so-*"
```

In quiet mode (`-q`) or non-interactive environments (CI), `b` will automatically pick the first match and emit a warning with a hint to use `--asset`.

## Flags

| Flag         | Description                               |
|--------------|-------------------------------------------|
| `--add`      | Add binary to b.yaml during install       |
| `--alias`    | Install binary under a different name     |
| `--asset`    | Glob pattern to filter release assets (e.g. `"argsh-so-*"`) |
| `--fix`      | Pin the specified version in b.yaml       |
| `-h`, `--help` | help for install                          |

## Global Flags

| Flag                 | Description                                                              |
|----------------------|--------------------------------------------------------------------------|
| `-c`, `--config string`  | Path to configuration file (current: `/home/fentas/github/fentas/b/.bin/b.yaml`) |
| `--force`            | Force operations, overwriting existing binaries                          |
| `-o`, `--output stringArray` | output options: json\|yaml\|format                                  |
| `-q`, `--quiet`      | Quiet mode                                                               |
| `-v`, `--version`    | Print version information and quit                                       |
